GDB-002H --- Les Histoires Émergentes

Version : 1.0 Statut : Officiel Type : Fondations du 
Gameplay
⸻
OBJECTIF

Définir le rôle des histoires émergentes dans Chroniques.

Les histoires émergentes constituent le cœur de l'expérience du joueur.

Elles remplacent la dépendance à un scénario linéaire.
⸻
DÉFINITION

Une histoire émergente naît naturellement de l'interaction entre :

	⁃	le joueur ;
	⁃	le monde ;
	⁃	les personnages ;
	⁃	les systèmes ;
	⁃	les conséquences.

Elle n'est pas écrite à l'avance.
⸻
LE JOUEUR EST L'AUTEUR

Chroniques ne cherche pas à raconter une histoire unique.

Le monde fournit les conditions.

Le joueur écrit son propre récit par ses décisions.
⸻
LES SYSTÈMES CRÉENT LES HISTOIRES

Les systèmes ne doivent pas uniquement produire des récompenses.

Ils doivent produire des situations.

Les situations donnent naissance aux souvenirs.
⸻
CHAQUE SESSION DOIT POUVOIR ÊTRE RACONTÉE

À la fin d'une session, le joueur doit pouvoir raconter ce qu'il a vécu.

Même une journée calme peut devenir une histoire.
⸻
L'IMPRÉVISIBLE

Les meilleures histoires apparaissent souvent lorsque plusieurs systèmes 
interagissent de manière inattendue.

Le game design doit favoriser cette émergence sans la forcer.
⸻
RÈGLES DE CONCEPTION

Toute mécanique devra :

	1.	favoriser les interactions entre systèmes ;
	2.	créer des situations plutôt que des scripts ;
	3.	laisser une place importante aux choix du joueur ;
	4.	produire des souvenirs mémorables ;
	5.	respecter la cohérence du monde.
⸻
CRITÈRE DE VALIDATION

Cette mécanique augmente-t-elle les chances qu'un joueur puisse 
raconter une histoire unique après sa session ?

Si la réponse est non, elle devra être repensée.
⸻
