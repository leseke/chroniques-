GDB-002D --- Les Événements Dynamiques

Version : 1.0
Statut : Officiel
Type : Fondations du Gameplay
⸻


OBJECTIF

Définir les principes qui gouvernent les événements dynamiques de 
Chroniques.

Les événements dynamiques permettent au monde de créer des situations 
nouvelles sans scénario imposé.
⸻
DÉFINITION

Un événement dynamique est une évolution naturelle du monde.

Il peut être local ou global.

Il apparaît parce que les conditions du monde le permettent, et non 
parce qu'un script décide qu'il doit se produire.
⸻
LE MONDE PREND L'INITIATIVE

Les événements peuvent apparaître sans intervention du joueur.

Le joueur choisit ensuite d'y participer, de les ignorer ou d'en tirer 
parti.

Le monde continue d'évoluer dans tous les cas.
⸻
PAS DE FOMO

Les événements enrichissent le monde.

Ils ne doivent jamais donner au joueur le sentiment d'avoir perdu une 
opportunité irremplaçable parce qu'il était absent.

Un événement disparu laisse naturellement la place à d'autres.
⸻
DIVERSITÉ

Les événements doivent varier par :

	⁃	leur ampleur ;
	⁃	leur durée ;
	⁃	leurs causes ;
	⁃	leurs conséquences.

Le joueur ne doit pas ressentir une répétition artificielle.
⸻
CONSÉQUENCES

Chaque événement peut modifier durablement :

	⁃	un lieu ;
	⁃	une communauté ;
	⁃	une économie ;
	⁃	un paysage ;
	⁃	une lignée.

Les conséquences dépendent toujours du contexte.
⸻
RÈGLES DE CONCEPTION

Toute mécanique d'événement devra :

	1.	naître d'une cause crédible ;
	2.	respecter le rythme du joueur ;
	3.	produire des conséquences cohérentes ;
	4.	éviter les scripts artificiels répétitifs ;
	5.	enrichir les histoires émergentes.
⸻
CRITÈRE DE VALIDATION

Cet événement donne-t-il l'impression que le monde vit de lui-même 
tout en respectant la liberté et le temps du joueur ?

Si la réponse est non, il devra être repensé.
⸻
