# GDB

## Rôle

La Game Design Bible (GDB) regroupe l’ensemble des spécifications fonctionnelles de Chroniques.

Elle décrit toutes les mécaniques de jeu, les systèmes, les interactions et les règles qui définissent l’expérience du joueur.

—

## Objectifs

- Décrire chaque fonctionnalité du jeu.
- Définir le comportement des systèmes.
- Servir de référence pour le développement.

—

## Contenu

- Gameplay
- Mécaniques
- Progression
- Économie
- Relations
- IA
- Interfaces
- Équilibrage

—

## Convention

Chaque document est identifié par un numéro unique.

Exemples :

- GDB-001
- GDB-002
- GDB-003

Les sous-sections utilisent une lettre.

Exemples :

- GDB-001A
- GDB-001B
- GDB-001C

—

## Dépendances

La GDB applique les décisions définies dans les documents MASTER.
