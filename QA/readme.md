# QA

## Rôle

Le dossier **QA** (Quality Assurance) regroupe toute la documentation liée à la qualité de Chroniques.

Il permet de définir les procédures de test, d’identifier les anomalies, de suivre leur résolution et de garantir la stabilité du jeu avant chaque version.

—

## Objectifs

- Garantir la qualité du jeu.
- Détecter et suivre les anomalies.
- Vérifier le bon fonctionnement des fonctionnalités.
- Assurer la stabilité des versions avant leur publication.

—

## Contenu

- Plans de test
- Cas de test
- Rapports de bugs
- Suivi des corrections
- Tests de régression
- Tests de performance
- Tests de compatibilité
- Validation des versions

—

## Convention

Chaque document est identifié par un numéro unique.

Exemples :

- QA-001
- QA-002
- QA-003

—

## Dépendances

Le contenu du dossier QA s’appuie sur les documents MASTER, les spécifications de la GDB et la documentation TECH afin de vérifier que les fonctionnalités sont conformes aux attentes.
