# MASTER

## Rôle

Le dossier **MASTER** contient les documents fondateurs de Chroniques.

Ces documents définissent la vision, les règles, l’architecture, les conventions et la feuille de route du projet.

Le contenu du dossier MASTER fait autorité sur l’ensemble du dépôt.

—

## Objectifs

- Définir la vision globale.
- Établir les principes de conception.
- Décrire l’architecture du projet.
- Fixer les conventions de documentation.
- Maintenir la roadmap officielle.

—

## Règles

- Les documents MASTER doivent rester concis et stables.
- Toute modification majeure doit être validée avant d’être intégrée.
- Les autres dossiers (GDB, LORE, TECH, etc.) doivent respecter les décisions prises dans MASTER.

—
